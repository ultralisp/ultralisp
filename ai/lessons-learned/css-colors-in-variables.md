# CSS-цвета и стили — выносить в переменные

**Проект:** Ultralisp (переход на Tailwind / reblocks-ui2)

## Проблема

Цвета ссылок (`text-sky-600 hover:text-sky-700`) были захардкожены в каждом `(:a :class ...)` по всему проекту — в 13 файлах, 34+ вхождения. При добавлении тёмной темы пришлось бы менять каждый файл.

## Решение

Объявить переменную в `src/variables.lisp` и использовать её везде:

```lisp
;; src/variables.lisp
(defvar *link-color-classes*
  "text-sky-600 hover:text-sky-700"
  ;; "dark:text-sky-400 dark:hover:text-sky-300"
  )
```

Пока что оставляй закомментированной часть с `dark:` префиксом, потому что Ultralisp пока не поддерживает тёмную тему.

Использование в виджетах:

```lisp
;; Простая ссылка — переменная как единственный класс
(:a :href url :class *link-color-classes* name)

;; Объединение с другими классами через join-css-classes
(:a :class (join-css-classes theme "dist enabled" *link-color-classes*)
    :href url
    name)
```

## Правило

Любой цвет или стиль, который используется более чем в одном месте, нужно вынести в `defvar` в `src/variables.lisp` (или рядом с виджетом, если это локальный стиль). Это позволяет:

- Добавить/изменить тему в одном месте
- Поддерживать тёмную тему через `dark:` префиксы
- Избежать расхождения стилей между страницами
